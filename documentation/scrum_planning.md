## **Scrum Planning Meeting**

During the Scrum planning session, the team met to define the project scope, epics, and sprint goals for the Job Tracker Application system. Key discussions included:

* **Finalizing Epics:**
    * E1: Authentication & Profiles UIs
    * E2: Item Management - Item Dashboard Implementation + Item CRUD Operations
    * E3: Messaging and Requests
    * E4: Profile Verification, Virtual Accounts
    * E5: Sorting, Filtering, Search Items
    * E6: Profile Ratings

* Establishing the initial tech stack — Ruby on Rails for backend, React for frontend, and Heroku for deployment.

* **Division of responsibilities**:

    * Alex : Messaging and Requests, Sorting, Filtering, Search Items
    * Hasitha: Login, Google OAuth, Image Uploads, User Ratings
    * Jingdong: Virtual Accounts, Seller User, Seller Rating Information, Item Information Display
    * Malvika: Item Dashboard and CRUD Operations on Item, User Profile, Profile Verification
    
* The team also agreed to use Git branching strategy (feature branches + pull requests) and maintain consistent Ruby version (3.3.4).
